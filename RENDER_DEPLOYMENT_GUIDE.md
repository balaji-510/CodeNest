# CodeNest Render Deployment Guide

## Prerequisites
- GitHub account with your CodeNest repository
- Render account (sign up at https://render.com)

## Deployment Steps

### 1. Push Your Code to GitHub
```bash
cd CodeNest
git add .
git commit -m "Add Render deployment configuration"
git push origin main
```

### 2. Deploy on Render

#### Option A: Using render.yaml (Recommended)
1. Go to https://dashboard.render.com
2. Click "New" → "Blueprint"
3. Connect your GitHub repository
4. Render will automatically detect `render.yaml` and create:
   - Backend web service
   - Frontend static site
   - PostgreSQL database

#### Option B: Manual Setup

**Backend Setup:**
1. Click "New" → "Web Service"
2. Connect your GitHub repository
3. Configure:
   - Name: `codenest-backend`
   - Root Directory: `codenest_backend`
   - Runtime: `Python 3`
   - Build Command: `./build.sh`
   - Start Command: `gunicorn codenest_backend.wsgi:application`

**Frontend Setup:**
1. Click "New" → "Static Site"
2. Connect your GitHub repository
3. Configure:
   - Name: `codenest-frontend`
   - Root Directory: `project2`
   - Build Command: `npm install && npm run build`
   - Publish Directory: `dist`

**Database Setup:**
1. Click "New" → "PostgreSQL"
2. Name: `codenest-db`
3. Copy the Internal Database URL

### 3. Configure Environment Variables

**Backend Environment Variables:**
Go to your backend service → Environment tab and add:

```
DJANGO_SETTINGS_MODULE=codenest_backend.settings_production
SECRET_KEY=<auto-generated-by-render>
DEBUG=False
DATABASE_URL=<from-database-connection>
ALLOWED_HOSTS=<your-backend-url>.onrender.com
CORS_ALLOWED_ORIGINS=https://<your-frontend-url>.onrender.com
EMAIL_HOST_USER=<your-email@gmail.com>
EMAIL_HOST_PASSWORD=<your-app-password>
GROQ_API_KEY=<your-groq-api-key>
OPENAI_API_KEY=<your-openai-api-key-optional>
GEMINI_API_KEY=<your-gemini-api-key-optional>
TEACHER_REGISTRATION_CODE=TEACHER2024
```

**Note:** Copy the actual values from your local `.env` file in `codenest_backend/.env`

**Frontend Environment Variables:**
Go to your frontend service → Environment tab and add:

```
VITE_API_URL=https://<your-backend-url>.onrender.com
VITE_ENABLE_CONTESTS=true
VITE_ENABLE_DISCUSSIONS=true
```

### 4. Update CORS After Deployment

After both services are deployed, update the backend's `CORS_ALLOWED_ORIGINS` with your actual frontend URL:

```
CORS_ALLOWED_ORIGINS=https://codenest-frontend.onrender.com
```

### 5. Verify Deployment

1. Visit your frontend URL
2. Test user registration/login
3. Test code execution features
4. Check that API calls work correctly

## Important Notes

- **Free Tier Limitations**: Render's free tier spins down after 15 minutes of inactivity. First request after spin-down may take 30-60 seconds.
- **Database Backups**: Free PostgreSQL databases are deleted after 90 days. Upgrade to paid plan for persistence.
- **Docker Features**: Code execution with Docker may not work on Render's free tier due to Docker-in-Docker limitations. Consider upgrading or using alternative code execution services.
- **Environment Variables**: Never commit sensitive keys to Git. Always use Render's environment variable management.

## Troubleshooting

**Build Fails:**
- Check build logs in Render dashboard
- Ensure `build.sh` has execute permissions: `chmod +x codenest_backend/build.sh`

**Database Connection Issues:**
- Verify `DATABASE_URL` is correctly set
- Check that `psycopg2-binary` is in requirements.txt

**CORS Errors:**
- Ensure frontend URL is in `CORS_ALLOWED_ORIGINS`
- Check that both URLs use HTTPS

**Static Files Not Loading:**
- Run `python manage.py collectstatic` manually
- Verify `STATIC_ROOT` is configured correctly

## Updating Your Deployment

Push changes to GitHub:
```bash
git add .
git commit -m "Your update message"
git push origin main
```

Render will automatically rebuild and redeploy your services.
